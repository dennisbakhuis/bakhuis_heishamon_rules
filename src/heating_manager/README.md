# Heating Manager — HA Dashboard for Panasonic Aquarea / HeishaMon

A Lovelace dashboard for monitoring your Panasonic Aquarea heat pump via [HeishaMon](https://github.com/Egyszeregy/HeishaMon). Displays live status, temperatures, efficiency (COP), rules state, and operational counters — all using built-in Home Assistant cards (no HACS required).

---

## What's in this folder

| File | Purpose |
|---|---|
| `dashboard.yaml` | The Lovelace dashboard — import this into HA |
| `sensors.yaml` | Optional MQTT & template sensors (room temp, standalone COP) |
| `automations.yaml` | Automation to sync room temperature to HeishaMon for RTC |

---

## Prerequisites

Before installing, ensure you have:

1. **HeishaMon** connected and publishing to your MQTT broker
2. **Home Assistant MQTT integration** set up and connected to the same broker
3. **HeishaMon HA integration** (`heishamon.yaml`) loaded in HA — this creates all the `sensor.aquarea_*` entities the dashboard uses

If you haven't imported `heishamon.yaml` yet, get it from the [HeishaMon repository](https://github.com/Egyszeregy/HeishaMon/blob/master/Integrations/Home%20Assistant/heishamon.yaml) and follow the instructions in its header.

---

## Step 1 — Verify your entity IDs

The dashboard uses entity IDs generated by the HeishaMon HA integration. HA derives entity IDs from sensor names, so they should match exactly — but double-check a few key ones:

1. In HA, go to **Settings → Devices & Services → Entities**
2. Search for `aquarea` — you should see sensors like:
   - `sensor.aquarea_power_state`
   - `sensor.aquarea_outlet_temperature`
   - `sensor.aquarea_heat_power_produced`
   - `sensor.heishamon_cop`
3. If your entities have a different prefix (e.g. `sensor.panasonic_aquarea_*`), do a find-and-replace in `dashboard.yaml` before importing.

---

## Step 2 — Add the dashboard to Home Assistant

### Option A: Via the UI (recommended)

1. In HA, go to **Settings → Dashboards**
2. Click **+ Add Dashboard**
3. Set a title (e.g. `Heating Manager`) and icon (`mdi:heat-pump`)
4. ✅ Enable **"Show in sidebar"** — this adds it to the left menu
5. Click **Create**
6. Open the new dashboard, click the **pencil icon** (Edit), then the **three-dot menu → Raw configuration editor**
7. Replace the entire contents with the content of `dashboard.yaml`
8. Click **Save**

### Option B: Via `ui-lovelace.yaml` (file-based config)

If you use `lovelace: mode: yaml` in `configuration.yaml`:

```yaml
# In configuration.yaml
lovelace:
  mode: yaml
  resources: []
  dashboards:
    heating-manager:
      mode: yaml
      title: Heating Manager
      icon: mdi:heat-pump
      show_in_sidebar: true
      filename: lovelace/heating_manager.yaml
```

Then copy `dashboard.yaml` to your HA config directory as `lovelace/heating_manager.yaml`.

## Step 3 — (Optional) Add room temperature sensors for RTC

The dashboard includes a Room Temperature section (Section 2) and a Rules State section (Section 5) that reference:
- `sensor.heishamon_room_temperature`
- `sensor.heishamon_room_setpoint`

These don't exist by default. To add them:

1. Open `sensors.yaml` and **uncomment** the MQTT sensor block
2. Add those lines to your `configuration.yaml` under `mqtt: sensor:`
3. Set up the automation from `automations.yaml` to publish your room sensor to MQTT (replace the placeholder entity IDs)
4. Restart Home Assistant

Until these sensors exist, the dashboard will show "Entity not available" for those cards — everything else still works.

---

## Step 4 — (Optional) COP template sensors

If you are using the full `heishamon.yaml` integration package, the COP sensors are already created:
- `sensor.heishamon_cop` — Combined COP (heat + DHW + cool)
- `sensor.heishamon_heat_cop` — Heat-only COP

The dashboard uses these directly. **No extra setup needed if you have heishamon.yaml.**

If you are NOT using `heishamon.yaml`, see the template sensor block in `sensors.yaml` and uncomment it.

## Dashboard sections explained

### Section 1 — Status Bar
A horizontal row of entity cards showing:
- **HP State** — On / Off
- **Mode** — Heat / DHW / Heat+DHW / Cool / Auto / etc.
- **Defrost** — Inactive or Active (a red alert banner appears when active)
- **Quiet Mode** — Off / 1 / 2 / 3 / Scheduled
- **Compressor Frequency** — Hz (indicates how hard the HP is working)

### Section 2 — Temperature Overview
Current values for all key temperatures:
- Outdoor ambient and outside pipe
- Inlet and outlet water temperatures
- Z1 heat request (target setpoint)
- Z1 water temperature
- Room temperature and setpoint (requires RTC setup — see Step 3)

### Section 3 — Temperature History Graph (24h)
A line graph showing trends for outlet, inlet, outdoor, and Z1 target temperatures over the last 24 hours. Useful for spotting weather compensation in action.

### Section 4 — Power & Efficiency
- Live heat power produced and consumed (W)
- Total combined power (heat + DHW + cool)
- Live COP (combined and heat-only)
- A markdown card explaining the COP reading
- A 24-hour power history graph (produced vs consumed)

> COP = heat energy out ÷ electricity in. A heat pump with COP 3.0 delivers 3 kWh of heat per 1 kWh of electricity.

### Section 5 — HeishaMon Rules State
Shows the output of the HeishaMon rules engine:
- **Z1 Heat Request Temperature** — this IS the calculated setpoint the rules produce
- Z1 Water Target temperature
- Room temperature and setpoint (if RTC is active)
- Pump flow, compressor frequency, and 3-way valve state (confirm the HP is responding)
- `input_number.heishamon_heatshift_z1` — the manual setpoint input from heishamon.yaml
- Heat mode and quiet mode selectors (interactive)

### Section 6 — Operational Stats
Lifetime counters and real-time metrics:
- Compressor operating hours and start/stop counter
- Pump flow and speed
- Fan 1 and Fan 2 speed
- Compressor current (A)
- Last error code

## MQTT topics reference

| Topic | Direction | Description |
|---|---|---|
| `panasonic_heat_pump/main/Heatpump_State` | HP → HA | Power state (0/1) |
| `panasonic_heat_pump/main/Operating_Mode_State` | HP → HA | Heat mode |
| `panasonic_heat_pump/main/Z1_Heat_Request_Temp` | HP → HA | Z1 setpoint / heat request |
| `panasonic_heat_pump/main/Heat_Power_Production` | HP → HA | Heat power produced (W) |
| `panasonic_heat_pump/main/Heat_Power_Consumption` | HP → HA | Electricity consumed (W) |
| `panasonic_heat_pump/opentherm/read/roomTemp` | HA → HP | Room temp for RTC |
| `panasonic_heat_pump/opentherm/read/roomTempSet` | HA → HP | Room setpoint for RTC |
| `panasonic_heat_pump/commands/SetZ1HeatRequestTemperature` | HA → HP | Override Z1 setpoint |

## Troubleshooting

**Entities show "unavailable"**
- Check that HeishaMon is connected and publishing to MQTT
- Verify the MQTT broker is reachable from HA
- Confirm `heishamon.yaml` is loaded (check Developer Tools → States for `sensor.aquarea_*`)

**Entity IDs don't match**
- HA generates entity IDs from sensor names on first load. If names were customised, IDs may differ.
- Check actual IDs in Settings → Entities → search `aquarea`
- Use find-and-replace in `dashboard.yaml` to update them

**Dashboard not in sidebar**
- Edit the dashboard in Settings → Dashboards, toggle "Show in sidebar"

**COP shows 0.0 or unavailable**
- COP is only meaningful when the HP is actively running
- Verify `sensor.heishamon_cop` exists (requires heishamon.yaml template sensors)
- If unavailable, the HP may not be producing heat at that moment
